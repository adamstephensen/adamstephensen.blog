<h1>We ALWAYS automate our deployments. We always deploy often. <br></h1> <p>This has many, many benefits. The most important of which is early feedback.</p> <p>An essential tool when managing deployment of many sites to different environments is a smoke-test page that validates the configuration of the website in that particular environment. </p> <p>SSW.HealthCheck makes it easy to check your site status.</p> <p> </p> <p><a href="./images/clip_image0164.jpg"><img title="clip_image016[4]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image016[4]" src="./images/clip_image0164.jpg" width="528" height="458"></a></p> <p>Figure: SSW.HealthCheck let’s us see at a Glance that on this site: the database connection is valid, but Debug mode has not been turned off and SMTP is not configured. We can also easily extend SSW.HealthCheck to perform other tasks like ensuring your database is up to date. I’ll address this in a future article.</p> <p> </p> <h1>Create a new web project</h1> <p>Lets start with a new web project, then we’ll add SSW.HealthCheck.</p> <p><a href="./images/clip_image0025.jpg"><img title="clip_image002[5]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image002[5]" src="./images/clip_image0025.jpg" width="560" height="317"></a> <p>Figure: Create a new web project called Northwind.MusicStore.WebUI <p>  <p><a href="./images/clip_image0045.jpg"><img title="clip_image004[5]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image004[5]" src="./images/clip_image0045.jpg" width="559" height="392"></a> <p>Figure: Select the MVC template and click OK.<br>Optionally: add the ‘Web.API’ and ‘Add unit tests’ options. <h5>Add SSW.HealthCheck.MVC5 to WebUI</h5> <p><a href="./images/clip_image0064.jpg"><img title="clip_image006[4]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image006[4]" src="./images/clip_image0064.jpg" width="553" height="116"></a> <p>Figure: Select Tools | Library Package Manager | Package Manager Console. Enter <b>Install-Package SSW.HealthCheck.MVC5</b> and press Enter. <p>Note: Alternatively you can use Tools | Library Package Manager | Manage NuGet Packages for the Solution, and then search for SSW.HealthCheck.MVC5 and click Install. <p><a href="./images/clip_image0084.jpg"><img title="clip_image008[4]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image008[4]" src="./images/clip_image0084.jpg" width="557" height="234"></a><br>Figure: Ensure SignalR is configured: Open Startup.cs from the root of your application. Ensure app.MapSignalR() is being called in the Configuration method. <p><a href="./images/clip_image0104.jpg"><img title="clip_image010[4]" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image010[4]" src="./images/clip_image0104.jpg" width="567" height="372"></a> <p>Figure: Run the application and navigate to /HealthCheck <h5>Configure HealthCheck to Test Your Connection Strings</h5> <p><a href="./images/clip_image012.jpg"><img title="clip_image012" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image012" src="./images/clip_image012.jpg" width="563" height="351"></a> <p><a href="./images/clip_image014.jpg"><img title="clip_image014" style="border-top:0;border-right:0;background-image:none;border-bottom:0;padding-top:0;padding-left:0;border-left:0;display:inline;padding-right:0;" border="0" alt="clip_image014" src="./images/clip_image014.jpg" width="565" height="385"></a><br>Figure: Because we are using Entity Framework Code First, and it does not actually create the database until the first time it is accessed: Click the <b>Log In</b> link from the toolbar, enter any username and password and click <b>Log In</b>.
